# 笔记风格

  • 用空格比逗号好
  • 用换行比句号好
  • 不用[xxx](http://)隐藏链接(除非太长) --> 网址本身也是一种信息
  • 不分h1-h6 --> 让工具自行分析 @lazy
  • 用分隔线(大波浪)适时地划分逻辑内容
  • 每篇文章最好有至少一张图片
  • 图片尽量用本地或自己的源, 减少引用第三方来源
  • 读书笔记不要大量摘抄, 主要记录感触最深的地方(先抄个目录逼自己一把)

# 目录层级

  • 最多9个chapter
  • 最多5个topic
  • appendix, 应该是补充内容(建议仅1个)
[0]collection 
    ch9-chapter 
        topic5 
            _appendix1

// appendix如下: 
private --> 私人内容, 不方便公开的
cmpt, case/demo, tricks, steps, tag --> 组件/案例/示例/技巧总结/步骤/标签
record, idea/faq, todo/plan, advice --> 按照时间轴记录
notnow/old, offcut --> 不推荐的/过时内容
code/gist/snippets, worm --> 代码片段/文章选段
spec --> 标准规范
corner/minor --> 少数派
